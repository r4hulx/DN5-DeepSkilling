# Git-HOL Lab 5: Remote Collaboration Workflow

This document records the remote-collaboration commands practiced in Lab 5.

## Commands practiced
- `git remote add origin <url>` — link a local repo to a remote
- `git clone <url>` — get a full copy of a remote repo (used to obtain this repo)
- `git push origin <branch>` — publish local commits
- `git pull origin <branch>` — fetch + merge remote changes
- `git fetch` — download remote history without merging
- `git checkout -b feature/git-hol-lab5-docs` — create a feature branch for this change
- Opening a Pull Request from `feature/git-hol-lab5-docs` into `main` on GitHub
- Merging the Pull Request (Gitflow / feature-branch workflow)
- Deleting the feature branch after merge (branch cleanup)

## Real PR raised for this lab
This exact branch was pushed to GitHub and opened as a real Pull Request against
`main`, then merged through the GitHub UI/API — demonstrating the fork/PR-style
collaboration workflow end-to-end rather than just documenting it.
